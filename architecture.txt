--- src/pages/Login.tsx ---
Exports: default Login
Imports (local): ../lib/auth (setAuth, getToken, getRole)
Component: Login
Description: Two-step login form ("credentials" → optional "2fa").
  On mount, if already authenticated, redirects CLIENT → /dashboard,
  others → /admin.
Fetch calls:
  POST ${API_BASE}/api/auth/login
    body: { email, password }
    response: { token?, user?, requires_2fa? }
    on requires_2fa=true → advances to 2FA step
    on token+user → setAuth(token, role), navigate by role
  POST ${API_BASE}/api/auth/verify-2fa
    body: { email, code }
    response: { token, user }
    on success → setAuth(token, role), navigate by role
  (API_BASE defined inline as import.meta.env.VITE_API_BASE ||
   "https://technicloudapi.onrender.com" — duplicates config.ts)
Role routing: CLIENT → /dashboard, all others → /admin.

Exports: default ProjectView
Imports (local): ../lib/auth (getToken), ../config (API_URL)
Internal components (not exported): MetricCard, DeviceTabBar, WaveformChart
Component: ProjectView
Description: SCADA-style real-time telemetry view for a single project.
  Features:
  - Dark/light theme toggle (independent of ThemeContext — local state)
  - Device tab bar (one tab per device_name in telemetry data)
  - Variable toggle pills to show/hide individual metrics
  - MetricCard grid — displays latest value with sparkline chart
  - WaveformChart — Recharts LineChart or data grid view with toggle
  - Export to Excel button
  Falls back to DEMO_ROWS if API is unreachable.
  Polls every 3000 ms using ?since=<lastTimestamp> for incremental updates.
  MAX_HISTORY = 120 points, SPARK_HISTORY = 20 points, POLL_MS = 3000.
Fetch calls:
  GET ${API_URL}/api/telemetry/${projectId}
    headers: Authorization: Bearer <token>
    response: { project_name?, rows: TelemetryRow[] } | TelemetryRow[]
    (initial load — replace=true)
  GET ${API_URL}/api/telemetry/${projectId}?since=<timestamp>
    headers: Authorization: Bearer <token>
    (polling — replace=false, appends new rows)
  GET ${API_URL}/api/export/${projectId}
    headers: Authorization: Bearer <token>
    response: Blob (xlsx)
    triggers browser download as project_<id>_export.xlsx

ROUTING & AUTH SUMMARY
================================================================================

Public routes:   /  /login  /careers  /projects
Protected routes (any authenticated user):
  /dashboard            — client project list
  /dashboard/:projectId — SCADA telemetry view
Admin-only routes (role must be MASTER or SUB_MASTER):
  /admin                — fleet + user + license management

Auth mechanism: JWT stored in localStorage["token"]; role in localStorage["role"].
On login, role determines redirect: CLIENT → /dashboard, others → /admin.
ProtectedRoute and AdminRoute guard client-side navigation.
